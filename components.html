Colors

A two-family surface system (Neutral grays + Brand blues) plus a shared text scale, all WCAG AA/AAA-vetted (per the Elastic library "Rule #1"). Brand colors (Elastic blue and yellow) sit alongside the surface families for chrome and accents. The thirteen legacy Search Labs tokens (--gray, --white, --border, etc.) are preserved as aliases that resolve to the new tokens — they keep existing CSS rendering while the prototype migrates to the new names.

Typography

Two type families. Headings use Mier B (with Inter as fallback); body copy and UI use Inter. Code samples use Roboto Mono. Sizes and line-heights are aligned to the Elastic library type scale — heading sizes round up slightly from the original Search Labs values, and line-heights tighten consistently.

✓ Mier B is loaded. All 10 weight/style combinations (Regular, Italic, Demi, DemiItalic, Bold, BoldItalic, ExtraBold, ExtraBoldItalic, Heavy, HeavyItalic) are self-hosted alongside design-system.css as .woff2 + .woff files. Loaded via @font-face with font-display: swap — text renders in Inter immediately, then swaps to Mier B when the font finishes loading. Headings now render in the proper brand typeface.

Font weights

TokenValueNotes
--fw-regular 400 (Inter Regular) Browser default; used where the body experiment doesn't apply. ~3 uses.
--fw-medium 500 (Inter Medium) UI labels, secondary headings, chip text, popular-item dates, link labels. ~17 uses. Library binds all Body / Label / Code styles at this weight.
--fw-semibold 600 (Inter SemiBold) Sub-headings (h4, h5), card titles, button labels (non-primary). ~17 uses.
--fw-bold 700 (Inter / Mier B Bold) Headings (h1–h3), primary buttons, emphasis. ~63 uses. ⚠ Deviation from library spec: library binds Display / Headline / Title styles at 800 (ExtraBold). We use 700 (Bold) instead — chosen for visual preference (ExtraBold read too heavy in our layout context). Mier B ExtraBold IS loaded and available; to switch to library spec, change --fw-bold: 700 → 800 in design-system.css. Session history: 700 (S15) → 800 (S19, library match) → 700 (S22, design choice).
--fw-body 500 (Inter Medium) 🧪 Experiment — applies only to the base body rule. Kept distinct from --fw-medium because it answers a different question ("what weight is body, specifically?" vs. "what weight is Inter Medium?"). Toggling this token reverts the body experiment without affecting any other 500-weight usage. Inline fallback <style> in index.html + components.html serves as a safety net if design-system.css fails to load.

Radius

Four corner-radius tokens. --r-4 is the workhorse — used for buttons, inputs, and most cards. --r-10 was consolidated to 8 px in the library alignment pass (no 10 px primitive exists upstream).

Blog card

The standard blog article card used across the prototype — blog landing, tag pages, article-related sections, author pages. Bare layout: no card chrome, sits directly on page background. Image gets its own hairline border and radius (--border + --r-8). Body is flush with the grid; vertical spacing handled by --s-16 gap between elements. Class name is .related-card (kept from earlier iterations to avoid sweeping ~1100 HTML class references).
